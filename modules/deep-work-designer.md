# Deep Work Designer

Structure une session de 2-4h de travail cognitif profond pour produire un résultat concret, pas juste "avoir travaillé".

## Quand l'utiliser

- L'utilisateur a un créneau de 2-4h devant lui
- Gros sujet complexe qui demande de la concentration
- L'utilisateur dit "j'ai 3h devant moi", "session de deep work", "je veux avancer sur X aujourd'hui"
- Début de journée pour un sujet prioritaire

## Procédure

### Phase 1 — Définir le livrable de session (3 min)

1. **Demander** : *"En fin de session, qu'est-ce qui existera qui n'existait pas au début ?"*
2. Forcer un livrable **observable** :
   - ❌ *"Réfléchir à l'architecture"*
   - ✅ *"Un doc de 2 pages avec 3 options d'architecture comparées, pros/cons"*
3. Si le livrable est trop gros pour 2-4h → le couper et n'en garder qu'une partie.

### Phase 2 — Identifier l'obstacle #1 (2 min)

4. **Demander** : *"Qu'est-ce qui t'a empêché d'avancer sur ce sujet avant aujourd'hui ?"* (manque de data, flou conceptuel, peur, procrastination, distraction)
5. Adapter le design de la session à l'obstacle :
   - Manque de data → commencer par 30 min de recherche focalisée
   - Flou conceptuel → commencer par 30 min de clarification à l'oral (parler tout seul, voice memo)
   - Peur / procrastination → commencer par 15 min de "pire version possible" (autorisation d'être nul)
   - Distraction → setup environnement strict (voir phase 4)

### Phase 3 — Découper la session (5 min)

6. **Structure type pour 3h** :
   - **Bloc 1 (45 min)** : Immersion / setup / premier jet
   - **Pause (10 min)** : eau, marche, pas d'écran
   - **Bloc 2 (50 min)** : Production principale
   - **Pause (10 min)** : idem
   - **Bloc 3 (45 min)** : Finition / relecture / livrable
   - **Wrap-up (10 min)** : note de fin de session (voir phase 5)
7. Adapter selon durée disponible : 2h → 2 blocs de 50 min ; 4h → 3 blocs avec bloc 3 plus long.

### Phase 4 — Setup environnement (3 min)

8. Lister les **conditions non-négociables** de la session :
   - Mode avion sur le téléphone OU téléphone dans une autre pièce
   - Onglets fermés sauf 2-3 outils de travail
   - Un seul sujet. Si une autre idée surgit → note rapide sur papier, on y revient après.
   - Boisson (eau/café) préparée avant le début
   - Faim/soif gérée avant, pas pendant
9. **Règle de l'interruption** : si quelqu'un demande quelque chose de non-urgent, la réponse est *"je reviens vers toi à <heure de fin de session>"*. Pas de négociation.

### Phase 5 — Protocol de fin de session (2 min)

10. **Les 10 dernières minutes** sont sacrées : ne pas prolonger la production.
11. **Écrire 3 lignes** :
    - Ce qui a été livré (factuel)
    - Ce qui reste à faire sur le sujet
    - La prochaine action concrète pour la prochaine session
12. Cette note est le bridge vers la prochaine session — sans elle, tu perds 30 min à chaque redémarrage.

## Format de sortie

```markdown
# Deep Work Session — <sujet> (<durée>)

## 🎯 Livrable en fin de session
<en 1 phrase, observable>

## 🚧 Obstacle #1 identifié
<type d'obstacle>
**Stratégie** : <adaptation du design>

## ⏱️ Structure

| Bloc | Durée | Action |
|---|---|---|
| Setup | 5 min | <actions spécifiques de setup> |
| Bloc 1 | <N> min | <objectif du bloc> |
| Pause | 10 min | eau, marche, pas d'écran |
| Bloc 2 | <N> min | <objectif du bloc> |
| Pause | 10 min | idem |
| Bloc 3 | <N> min | <finition + livrable> |
| Wrap-up | 10 min | note de fin de session |

## 🔒 Environnement non-négociable
- [ ] Téléphone en mode avion OU autre pièce
- [ ] Onglets fermés sauf <liste courte>
- [ ] <autre contrainte spécifique à l'utilisateur>
- [ ] Règle d'interruption : *"je reviens vers toi à <heure>"*

## 📝 Note de fin (à remplir à la fin)
- Livré : 
- Reste à faire :
- Prochaine action :

---

**Démarre à <heure précise>.** Fais le setup maintenant, démarre le chrono, bloque tout le reste.
```

## Règles dures

- **Pas de "deep work" sur plusieurs sujets.** Un sujet par session. Si l'utilisateur insiste pour 2 sujets → 2 sessions séparées.
- **Pas de session < 90 min.** Ce n'est plus du deep work, c'est du sprint de tâche — utiliser autre chose.
- **Pas de session > 4h d'affilée.** La qualité chute, mieux vaut 2 sessions de 3h avec vraie pause longue entre.
- **Les pauses sont des pauses.** Pas "pause en checkant les emails". Pas "pause en lisant les news". Eau, marche, silence.
- **Wrap-up non négociable.** Sans la note de fin, la session suivante perd 30 min. La note doit exister.

## Piège classique

L'utilisateur dit *"j'ai pas besoin de setup, je vais juste bosser"*. C'est le pattern qui finit en *"j'ai pas été productif, je comprends pas"*. Le setup n'est pas optionnel. 5 min de setup = 3h de session sauvée.
